# Decision Lifecycle

## 🧠 States

1. Draft
2. Clarification
3. Reaction
4. Objection
5. Revision
6. Adopted

---

## 🔄 Rules

* Transitions must follow order
* Only "Objection" phase allows blocking feedback
* Revision required if objection is valid
* Decision is adopted when no objections remain

---

## ⚠️ Constraints

* No skipping states
* No objection outside objection phase
* Only authorized users can interact

---

## ⏱ Deadlines (future)

* Each phase may have a deadline
* Automatic transitions possible
