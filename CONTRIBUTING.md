# CONTRIBUTING

## 🎯 Philosophy

* Keep code simple and readable
* Respect architecture (no business logic in controllers)
* Prioritize maintainability

---

## 🧩 Project Structure

* Models → data structure
* Services → business logic
* Controllers → HTTP layer (thin)
* Policies → authorization

---

## 🧪 Testing

* Focus on service layer
* Write meaningful tests (business logic)
* Avoid testing Laravel internals

---

## 🔧 Workflow

1. Create issue / pick task
2. Write tests (if relevant)
3. Implement feature
4. Submit PR

---

## ⚠️ Rules

* No spaghetti code
* No hidden logic
* Explicit naming only

---

## 💡 Tips

* Keep functions small
* Think in responsibilities
* Prefer clarity over cleverness
